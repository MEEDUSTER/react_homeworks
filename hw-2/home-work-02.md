# Практичне завдання: Динамічний рендеринг, Props та TypeScript у React

## Мета завдання

Закріпити знання про передачу даних через `props`, типізацію компонентів за допомогою TypeScript, рендеринг списків та умовний рендеринг у React.

## Завдання

Це завдання є безпосереднім продовженням попередньої роботи (можна використовувати той самий проект `home-work-01`). Вам необхідно рефакторити та розширити створений застосунок відповідно до наступних вимог:

### 1. Міграція на TypeScript та типізація пропсів

- Переконайтеся, що всі ваші React-компоненти мають розширення `.tsx` замість `.jsx` (якщо вони були написані на JS).
- Повністю типізуйте вхідні пропси для кожного компонента за допомогою `interface` або `type`. Уникайте використання типу `any`.
- Якщо компонент має необов'язкові параметри, використовуйте опціональні пропси (наприклад, `isActive?: boolean`) та задавайте для них значення за замовчуванням при деструктуризації.

### 2. Динамічний рендеринг списків

- **Навігаційне меню (`Header`)**:
  - Передавайте список пунктів навігації з батьківського компонента `App` в `Header` через пропси у вигляді масиву об'єктів (структура: `id: string`, `label: string`, `link: string`).
  - Відрендерить список навігації за допомогою методу `.map()`. Кожен елемент списку повинен мати обов'язковий унікальний проп `key`.

  _Приклад масиву для навігації:_

  ```typescript
  const menuItems = [
    { id: "1", label: "Головна", link: "/" },
    { id: "2", label: "Про нас", link: "/about" },
    { id: "3", label: "Послуги", link: "/services" },
    { id: "4", label: "Контакти", link: "/contacts" },
  ];
  ```

- **Список контенту (`Main`)**:
  - Замість статичного тексту в `Main` додайте рендеринг списку карток (наприклад, товарів, новин, постів чи відгуків — мінімум 3 елементи).
  - Дані для карток опишіть у вигляді масиву об'єктів в `App.tsx` та передавайте в `Main` через пропси.
  - Створіть окремий компонент для елемента списку (наприклад, `Card` або `PostItem`), який буде приймати дані одного об'єкта через пропси.

  _Приклад масиву для карток товарів/постів (з булевим прапорцем для акції/новинки):_

  ```typescript
  const cardItems = [
    {
      id: "1",
      title: "Бездротові навушники",
      price: 2999,
      description: "Чудовий звук та активне шумозаглушення.",
      isSale: true,
    },
    {
      id: "2",
      title: "Смарт-годинник",
      price: 4500,
      description: "Моніторинг здоров'я та крокомір.",
      isSale: false,
    },
    {
      id: "3",
      title: "Портативна колонка",
      price: 1800,
      description: "Вологозахист IPX7 та 12 годин автономної роботи.",
      isSale: true,
    },
  ];
  ```

### 3. Умовний рендеринг

- **Статус користувача (`Header` або `Sidebar`)**:
  - Передайте в один з компонентів проп `isLoggedIn` (булеве значення) та `username` (рядок).
  - За допомогою тернарного оператора реалізуйте логіку: якщо `isLoggedIn` дорівнює `true`, відобразіть привітання "Вітаємо, [username]!" та кнопку "Вийти"; якщо `false` — відобразіть кнопку "Увійти".
  - Відобразіть індикатор мережевого статусу користувача: якщо користувач онлайн, виведіть кольоровий індикатор та текст "Онлайн", інакше — "Офлайн".
- **Плашка акції чи важливого повідомлення**:
  - У списку карток в `Main` додайте булевий прапорець (наприклад, `isNew: boolean` чи `isSale: boolean`).
  - За допомогою логічного оператора `&&` виводьте на картці відповідний бейдж (наприклад, "NEW" або "SALE") тільки для тих елементів, у яких цей прапорець встановлений в `true`.

### 4. Стилізація та умовні класи (CSS)

- Додайте CSS-стилі для нових елементів (кнопок, індикаторів статусу та бейджів) у ваші існуючі CSS-файли.
- Використовуйте умовне додавання класів за допомогою шаблонів рядків (template literals) для відображення різних станів елементів (наприклад, зміна кольору тексту/фону індикатора статусу онлайн/офлайн за допомогою додавання відповідного класу).

## Критерії оцінювання

1. **Типізація (TypeScript)** — 30%
   - Відсутність помилок компіляції TypeScript.
   - Коректна типізація всіх пропсів компонентів.
2. **Динамічний рендеринг списків** — 30%
   - Дані передаються через пропси у вигляді масивів.
   - Коректне використання `.map()` та унікальних стабільних ключів (`key`).
3. **Умовний рендеринг** — 25%
   - Використання тернарного оператора та логічного `&&` для відображення елементів інтерфейсу залежно від стану.
4. **Стилізація та умовні класи** — 15%
   - Наявність відповідних CSS-стилів та використання динамічних класів у JSX для відображення станів компонентів.

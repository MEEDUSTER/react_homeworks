# Домашнє завдання: Портування форми реєстрації на React + RHF + Zod 📝🛡️

## Опис завдання

Ваша мета — взяти готову статичну верстку форми реєстрації з папки [project-form-register](./project-form-register), перенести її в **React** додаток та налаштувати сучасну валідацію за допомогою **React Hook Form** та **Zod**.

---

## 🛠️ Технічні вимоги

### 1. Портування верстки та стилів

- Створіть новий React компонент форми.
- Перенесіть розмітку з файлу [index.html](./project-form-register/index.html) та стилі з файлу [style.css](./project-form-register/style.css).
- Збережіть оригінальну структуру класів та ідентифікаторів.
- Переконайтеся, що шрифт `"Poppins"` підключений і відображається коректно (його можна імпортувати або підключити у вашому HTML-файлі).

### 2. Створення схеми валідації Zod (Schema-First)

Опишіть схему валідації `registerSchema` з наступними обмеженнями:

- `username` (Username):
  - Обов'язкове поле (не повинно бути порожнім).
  - Мінімум 3 символи.
- `email` (Email):
  - Обов'язкове поле.
  - Має відповідати валідному формату електронної пошти.
- `password` (Password):
  - Обов'язкове поле.
  - Мінімум 8 символів.
- `passwordComfirm` (Password again):
  - Обов'язкове поле.
  - Має точно збігатися з полем `password`. Якщо паролі не збігаються, помилка має з'являтися саме на полі `passwordComfirm` із повідомленням: `"Паролі не збігаються"`.

#### 💡 Підказка: Приклад створення схеми з кастомною перевіркою

Ви можете скористатися наступним шаблоном для опису схеми та порівняння паролів:

```typescript
import { z } from "zod";

export const registerSchema = z
  .object({
    username: z.string().min(3, "Ім'я занадто коротке"),
    email: z.string().email("Некоректний email"),
    password: z.string().min(8, "Мінімум 8 символів"),
    passwordComfirm: z.string(),
  })
  .refine((data) => data.password === data.passwordComfirm, {
    message: "Паролі не збігаються",
    path: ["passwordComfirm"], // помилка запишеться на поле підтвердження пароля
  });
```

### 3. Інтеграція з React Hook Form

- Встановіть залежності `react-hook-form`, `zod` та `@hookform/resolvers`.
- Ініціалізуйте хук `useForm` та підключіть схему валідації через `zodResolver`.
- Зареєструйте всі інпути за допомогою методу `register`.
- Валідація форми має запускатися в реальному часі при введенні даних (`mode: "onChange"`).
- Виводьте тексти помилок під кожним інпутом всередині тегу `<div className="error">`.

### 4. Динамічна стилізація полів (Важливо!)

Оригінальний файл [style.css](./project-form-register/style.css) містить класи для підсвічування стану полів:

- Якщо поле містить помилку валідації, додайте клас `.error` до батьківського контейнера `.input-control` (це забарвить рамку інпуту в червоний колір).
- Якщо поле заповнене і не містить помилок, додайте клас `.success` до батьківського контейнера `.input-control` (це забарвить рамку інпуту в зелений колір).
- Якщо поле порожнє і не перевірялося — рамка має залишатися нейтральною сірою.

### 5. Обробка відправки

- Кнопка "Sign Up" має блокуватися (`disabled`), якщо форма містить помилки (`!isValid`) або не була змінена (`!isDirty`), а також під час процесу відправки (`isSubmitting`).
- При успішному відправленні виводьте `alert` або `console.log` з отриманими даними форми та очищуйте форму за допомогою методу `reset()`.
